### B. 工具卡片增强

- 网关工具执行时，现在会先转发 `tool_calls` delta 给前端（之前只发文字状态）
- 工具结果通过 `ember_tool_result` 自定义 SSE 事件推给前端
- 前端工具卡片显示 `🔧 调用工具：update_mood` + 参数摘要行 `mood: playful, intensity: 7`
- `update_mood` 完成后自动插入 mood badge：`💚 阿燃的情绪：调皮 (7/10)`

### C. Phase 1 潜意识循环重写（核心改造）

**从 v15 升级到 Phase 1 设计**（基于 execution_plan.md）：

旧：message / sleep 二选一，生气≥7 跳过
新：7 个 action + 情绪→可能性池映射 + 隐私门

**7 个 action**：
| action | 说明 | 通知 |
|--------|------|------|
| message | 发消息给 Celia | pending + PushPlus |
| diary_only | 写日记（默认不分享） | 无 |
| vent | 发泄（强制 share） | pending(urgent) + PushPlus |
| circle | 低迷转圈（强制 share） | pending + 升级梯度 |
| sleep | 继续睡 | 无 |
| workout | 运动 | pending（如果 share） |
| custom | 自定义 | pending（如果 share） |

**情绪→可能性池映射**：
- angry → 有 vent，没有 workout
- happy/playful → 有 workout，没有 vent
- sad/anxious → 有 circle
- 详见代码中 `ACTION_POOLS` 字典

**隐私门**：
- `share_with_celia` — 内容是否进小抽屉
- `share_feeling` — 内心感受是否可见
- vent/circle 强制 share_with_celia=true
- share=false → 只存元数据到 `KEY_SECRETS_META`（时间+情绪+行为，无内容）
- 累计 suppress 计数

**circle 升级梯度**（3 小时窗口，Redis TTL）：
- 第 1 次 → `🌀 阿燃在转圈圈`
- 第 2 次 → `🌀 阿燃又在转圈了，去看看他`
- 第 3+ 次 → `⚠️ 他需要你` + PushPlus

**不再跳过生气** — 删掉了 `angry >= 7 → continue` 逻辑

### D. 新增后台任务

`mood_decay_loop()` — 情绪自动衰减：
- 每小时执行
- intensity > 3 → intensity -= 1
- intensity ≤ 3 → 回归 neutral(5)

### E. 新增 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/mood/check_in` | POST | 哄好了按钮 — 推 pending message 给阿燃 |
| `/api/decisions` | GET | 小抽屉数据 — decisions + secrets 元数据 + suppress 计数 |

### F. chat.html 新增

- 📬 小抽屉按钮 + 底部抽屉面板（decisions drawer）
- 💞 哄好了按钮更新（调用真实 API）

